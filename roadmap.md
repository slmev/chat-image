# AI 图片生成应用 - 客户端化开发路线图

## 当前状态

项目正在从 Web 应用迁移为 Tauri 桌面客户端，核心客户端化框架已经搭好：

- 已接入 Tauri 2 桌面壳层。
- 已实现 Web/Tauri 双运行时判断。
- 已抽象图片持久化接口。
- Tauri 运行时已支持将生成图片保存到 AppData/images。
- 元数据已从 localStorage 迁移到 Tauri Store。
- 桌面构建链路当前可用。

当前开发重点不再是 Web 功能扩展，而是把桌面端的数据可靠性、本地文件管理、导入导出和发布体验补完整。

---

## P0: 本地保存可靠性

### 1. 修复 Tauri 元数据保存竞态

- [ ] 将聊天记录保存改为可等待的异步流程。
- [ ] 将历史列表和历史消息保存改为可等待的异步流程。
- [ ] 将配置保存改为可等待的异步流程。
- [ ] 避免生成图片后立即退出应用导致最后一次 Store 写入丢失。
- [ ] 统一处理保存失败提示，不能静默丢数据。

涉及重点：

- `src/stores/chat.ts`
- `src/composables/useHistory.ts`
- `src/stores/config.ts`
- `src/platform/metadataStore.ts`

验收标准：

- 生成图片后立即重启应用，最新消息和图片仍存在。
- 收藏、删除、导入、清空等操作重启后状态一致。
- `npm run build:desktop` 通过。
- `npm test` 通过。

### 2. 完善本地图片生命周期管理

- [ ] 删除单条消息时，清理该消息中不再被其他记录引用的本地图片。
- [ ] 删除历史对话时，清理该历史记录独占的本地图片。
- [ ] 清空历史时，清理对应 AppData/images 图片。
- [ ] 删除失败时保留元数据操作结果，但给出可感知错误提示。
- [ ] 避免误删仍被其他消息或历史引用的图片。

涉及重点：

- `src/platform/tauriImageRepository.ts`
- `src/composables/useHistory.ts`
- `src/stores/chat.ts`

验收标准：

- 删除消息后，对应本地图片文件被清理。
- 删除历史后，对应本地图片文件被清理。
- 多处引用同一图片时不会误删。
- Web 运行时行为不受影响。

### 3. 增加孤儿图片清理能力

- [ ] 扫描 Store 中所有消息和历史记录引用的 `localPath`。
- [ ] 扫描 AppData/images 下实际存在的图片文件。
- [ ] 找出未被引用的孤儿图片。
- [ ] 提供手动清理入口。
- [ ] 清理前显示图片数量和预计释放空间。

验收标准：

- 能正确统计本地图片数量和占用空间。
- 能安全删除孤儿图片。
- 清理后现有历史图片仍能正常预览。

## P1: 桌面导入导出

### 4. 桌面导出改为 zip 包

- [ ] Web 端继续保留当前 JSON 导出。
- [ ] Tauri 端导出 zip 包。
- [ ] zip 内包含 `history.json`。
- [ ] zip 内包含 `images/` 目录。
- [ ] `history.json` 中图片引用使用相对路径，不使用当前机器的 AppData 绝对路径。
- [ ] 导出时通过系统保存对话框选择保存位置。

验收标准：

- 导出的 zip 在另一台机器或清空数据后仍可导入。
- zip 内 JSON 不包含大体积 base64。
- zip 内图片数量与历史记录引用一致。

### 5. 桌面导入支持 zip 包

- [ ] 读取并校验 `history.json`。
- [ ] 将 zip 内图片写入当前机器 AppData/images。
- [ ] 重写导入消息中的 `localPath`。
- [ ] 支持 replace 和 merge 两种模式。
- [ ] 导入失败时不破坏当前数据。

验收标准：

- replace 模式会替换当前历史。
- merge 模式只追加不重复的消息。
- 导入后重启应用，图片仍能正常显示。
- 图片缺失或 zip 损坏时给出明确错误提示。

## 技术栈

| 用途 | 方案 |
|------|------|
| 前端框架 | Vue 3 + TypeScript |
| 构建工具 | Vite |
| 状态管理 | Pinia |
| 桌面框架 | Tauri 2 |
| 元数据存储 | Tauri Store |
| 图片文件存储 | AppData/images |
| 文件能力 | Tauri fs/dialog/opener plugins |
| 网络能力 | 浏览器 fetch / Tauri http plugin |
| 测试 | Vitest |
